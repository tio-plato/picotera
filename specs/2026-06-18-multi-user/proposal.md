# 多用户功能

为 PicoTera 加入多用户与身份鉴权能力。第一期只引入用户与身份模型、HTTP Header 身份提供商、单用户模式，以及内部 API 的鉴权；不做细粒度权限控制，也不做资源的用户归属。

## 需求

1. **用户表**：用户 ID 为自增数字，另有显示名、是否为全局管理员两个字段。
2. **用户-身份提供商映射表**：每个用户可以绑定一个或多个身份提供商，每条映射记录身份提供商名称及该提供商下的唯一识别字段。
3. **HTTP Header 身份提供商**：第一期只支持 `http-header` 一种提供商，通过环境变量开启；用环境变量配置 header 名字。当请求携带该 header 时，按其值匹配用户。
4. **自动创建用户**：增加一个环境变量，开启后若匹配不到用户则自动创建一个新用户。
5. **单用户模式**：增加一个环境变量，开启后无论请求提交了什么 header，都识别为 `single-user-mode` 提供商、身份字段为 `root`。
6. **命令行管理员工具**：增加一个命令行子命令，将某个用户设置为管理员。
7. **鉴权边界**：网关路由不需要用户鉴权（这些路由使用 API Key 鉴权）；用户界面、PicoTera 内部 API 则需要鉴权。
8. **路由迁移**：原有的 unified 网关路由从 `/api/picotera` 迁移到 `/api/unified`，避免鉴权配置冲突。
9. **自身信息 API 与控制台展示**：增加获取自己用户信息（用户名、ID）的 API，并显示在控制台左下角；将原来的两个按钮靠右显示，腾出左边位置展示用户名。
10. 暂时不做复杂的用户管理功能。
11. 暂时不做权限控制，也不做资源的用户归属，后续再加。

## 补充确认（规划期间澄清）

- **CLI 定位用户**：通过用户 ID 定位，命令形如 `picotera set-admin <user-id>`。
- **单用户模式 root 用户**：开启单用户模式时，若 `(single-user-mode, root)` 身份无对应用户，自动创建该用户并设 `is_admin=true`，开箱即用且拥有全部权限；不受「自动创建用户」开关影响。
- **未配置任何身份提供商时**：`header` 与单用户模式都关闭时，所有 `/api/picotera` 内部 API 一律返回 401，必须显式配置至少一种提供商。
- **鉴权范围**：鉴权仅作用于 `/api/picotera`（内部管理 API，含 `test/direct`）；网关 catch-all 与迁移后的 `/api/unified` 不做用户鉴权。
