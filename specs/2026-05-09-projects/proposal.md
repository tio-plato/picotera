# Projects feature

增加一个"项目"功能。用户可以创建指定名字的项目，并配置多个项目路径。在 picotera 收到请求的时候，对整个请求体使用数个固定的正则表达式，尝试将项目路径匹配出来（匹配出来之后需要前后加引号然后按 JSON 反转义）。如果成功提取项目路径，则对数据库中的项目路径依次进行前缀匹配，找到匹配最长的那一条，记录为该请求的项目。在请求页面增加显示项目列。在追踪界面也增加显示项目列。在项目表中记录项目的最早出现时间和最后出现时间，就如同对 trace 做的那样。为了避免每次遍历数据库，对项目表，也增加一个内存缓存，就像对路由做的那样。更新 CLAUDE.md 使得此后我们不会忘记这点。

## 用户澄清后的细节

- 存储模型：`project` 行内 `paths JSONB` 数组（不引入独立 `project_path` 表）。
- 时间戳更新：请求命中项目时，同步对 project 行做 LEAST/GREATEST upsert（与 `UpsertTrace` 一致）。
- 抽取正则（按顺序，写死在 Go 源码内）：
  1. `Workspace root folder: (.*?)\n`
  2. `Primary working directory: (.*?)\n`
  3. `<cwd>(.*?)</cwd>`
- 抽取后处理：对每个 capture group，前后加 `"`，再以 JSON string 解码（处理 `\"`、`\\`、`\u00XX` 等转义）。
- 列展示：请求页与追踪页都加"项目"列；列头带 `ColumnFilter` 下拉过滤；点击单元格跳到带筛选条件的请求列表。
