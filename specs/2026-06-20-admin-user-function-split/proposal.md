# 前端功能的管理员 / 用户划分

把控制台功能划分为「用户功能」与「管理员功能」两类，在后端强制鉴权、在前端按当前用户权限自动隐藏无权访问的界面。

## 需求

1. **功能分类**：
   - **用户功能**：概览、密钥、请求、追踪、测试（仅网关测试部分）。
   - **管理员功能**：渠道、模型、端点、项目、脚本、缓存（KV）、汇率、用户、设置、模拟、测试中的短路测试。
2. **接口鉴权**：管理员功能相关的管理 API 仅允许 `is_admin=true` 的用户访问，非管理员请求返回 403。
3. **`/me` 暴露管理员标识**：`/me` 已返回 `isAdmin`，前端据此把侧栏导航分为「用户功能」「管理功能」两栏，非管理员自动隐藏「管理功能」一栏及其页面。
4. **测试拆分**：测试页面对所有登录用户开放；其中**短路测试**（`/api/picotera/test/direct`，绕过网关直连上游）为管理员功能，**网关测试**（经 API Key 走 `/api/unified` 或网关路径）为所有用户可用。

## 补充确认（规划期间澄清）

- **共享配置资源的访问策略**：渠道、模型、端点、项目这四类被划为管理员功能的资源，其列表数据被用户功能依赖（请求过滤、概览图例、网关测试下拉、追踪标签）。采用「**完全锁定 + 精简标签接口**」：完整 CRUD 列表/读写接口仅管理员可访问；另为用户功能新增只返回 `id`+名称（端点额外含 `path`、`endpointType`）的轻量标签接口，对所有登录用户开放。渠道的 `credentials` 等敏感字段只存在于管理员的完整接口，不出现在标签接口。
- **模拟（simulate）归类**：模拟改为**管理员功能**（原需求列为用户功能，此处修正）。模拟会返回候选渠道名、合并注解、出站请求 URL/凭据解析方式等路由内幕，归管理员更合适，且无需为其设计脱敏层。
