# Proposal: 端点"模型列表"类型

为端点增加一个"模型列表"（`modelList`）类型。

## 端点配置

- 选中"模型列表"类型时，无需输入"模型字段路径"（`modelPath` 为空）。
- 这种类型的端点无法绑定渠道（`provider_endpoint`）或渠道模型（`provider_models` 的 `endpoints` 过滤不在本需求范围内，仅限制 `provider_endpoint` 绑定）。

## 网关请求处理

当有该端点的请求时，采用单独的 `handleModelList` 方法处理：

1. 检查请求 Method 是否为 GET 或 HEAD，如果不是则返回 404。
2. 解析凭证（认证客户端）。
3. 对系统内所有的模型运行模拟：检查每个模型是否有至少一个可用的上游（非禁用的渠道、非禁用的模型、渠道有凭证、渠道端点有上游 URL）。
4. 将模拟通过（没报错且有可用上游）的模型记录下来。
5. 返回模型列表：`data[].id` 为模型名，其它字段不返回。

## 缓存

缓存先不做。
