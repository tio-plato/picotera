# 管理员全局概览页面

做一个管理员的全局概览页面。这个页面和个人用户的概览页面稍有不同：

1. 它可以看全局所有用户的统计数据（不限于当前管理员自己的数据）。
2. 它不能按密钥、项目筛选，这两个筛选被「用户」筛选替代。
3. API 方案由实现者决定（全新一套或复用现有端点加参数）。

## 补充确认（规划期澄清）

- **筛选项**：保留模型、上游模型、Provider、时间范围、币种筛选；去掉密钥与项目筛选；新增「用户」筛选。
- **图表分组维度**（环形图 / 时间序列 / 速度图 / Sankey）：维度集合为 `user`（新增，全局主维度）、`model`、`upstreamModel`、`provider`；去掉 `apiKey`、`project`。
- **API 方案**：新建 admin 专用端点（位于 `/api/picotera/admin/overview/*`，由 admin Huma 分组的 `requireAdmin` 中间件统一鉴权），不复用现有用户端点加参数，避免越权与共享查询的条件分支。
