# 用户管理功能

为 PicoTera 增加用户管理能力（API 与控制台界面），建立在已有的多用户与身份鉴权基础之上。管理员仍仅作标记，不做任何权限控制，也不做资源的用户归属。

## 需求

1. **用户增删改**：支持新建用户、删除用户、编辑用户。可编辑显示名与是否为管理员。
2. **身份管理**：支持查看用户关联的 identity provider 及对应 identity，并支持编辑、删除对应条目。
3. **启用 / 禁用用户**：支持禁用和启用用户。被禁用的用户无法通过鉴权（内部管理 API 返回 401）。
4. **删除级联**：删除用户时，一并删除其全部关联 identity。
5. 管理员仅作标记，不做任何权限控制。

## 补充确认（规划期间澄清）

- **自我锁定**：不对「当前登录用户操作自己」做任何限制。管理员可以禁用、删除自己或取消自己的管理员标记；后果由操作者承担（包括单用户模式下禁用 root 会导致全员 401）。
- **新建用户与身份录入**：采用两步式。新建用户表单只填显示名与是否管理员；建好后在用户的「身份」侧栏面板中单独增删改身份条目（参考渠道页面「端点绑定」面板的交互）。
- **禁用语义**：禁用是 `app_user` 上的标志位，由鉴权解析器统一检查，对所有身份提供商（含单用户模式 root）一致生效。
- **鉴权范围不变**：管理用户的 API 位于 `/api/picotera` 前缀，沿用既有用户鉴权中间件；任意已鉴权用户均可管理用户（不做 admin 门禁），与第一期约定一致。
