# Proposal: 在模型列表界面增加上游信息

在模型列表界面：

1. 对每个模型，增加上游数量显示；
2. 对每个模型增加一个操作按钮，点击后可以列出所有相关联的上游，及上游模型名；
3. 在列表页读取未被添加的模型（即有上游、但没有出现在模型列表里），并显示在尾部（点击展开），显示后操作按钮可以点击增加，即可打开新增模型对话框并预填模型名。

## 澄清

- 「上游数量」按渠道（provider）计数：`provider.providerModels` 中包含该模型 key 的 provider 数量。一个 provider 算一个上游，无论它绑定了多少端点。
- 上游列表面板字段：渠道名、上游模型名（缺省回退到本地模型名）、端点列表（entry.endpoints；为空时展开为该 provider 已绑定的全部端点路径）、可选优先级。
- 「未被添加的模型」入口：操作按钮打开 ModelForm 并预填名称，name 字段在该模式下只读（强制与上游一致）。
