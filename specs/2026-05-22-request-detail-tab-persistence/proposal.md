# 请求详情页：切换请求时保持 Tab 与展开态

在请求查看页面 (`RequestDetailView` / `RequestDetailsPanel`) 查看请求时：

- 当切换 span 卡片（在同一个请求详情视图内、或切换到另一个 request 详情）时，**【概览 / 原始请求 / 原始响应 / 日志】** 这一层主 Tab 的选中态需要保持不变，不要回退到「概览」。
- 同样地，在【原始请求】、【原始响应】内部，**【Raw / JSON / 聚合 / Events / 渲染】** 这一层 body Tab 的选中态也需要保持不变。
- 【原始请求】、【原始响应】内的 **Headers `<details>` 展开态** 也需要保持不变。

每个 Tab 的初始默认值仍按当前行为（概览 / 优先 JSON / Headers 折叠）。当因数据形态变化导致当前选中项不在可选项里时（例如响应从 SSE 变成 JSON 而当前选中 Events），自动回退到一个合法选项；除此之外不要因为数据变化主动重置用户的选择。

跨会话持久化不在范围内。
