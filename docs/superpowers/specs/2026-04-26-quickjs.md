# QuickJS 集成

库用这个纯 go 的： https://pkg.go.dev/modernc.org/quickjs

## 目标

设计一组记录在数据库中的 js 脚本。脚本可以绑定到渠道、端点、模型，或是模型x渠道x端点的映射，或是作为全局处理脚本。每条都可以绑定多个。

对每个客户端请求（或称之为元请求、下游请求），都将初始化一个 quickjs 上下文，这个上下文会一直存在，直到请求完整结束。

初始化上下文之后，首先加载一套固定的 sdk js 到上下文中。这套 sdk 使用 tapable 提供一套 hook 框架，提供一组注入点，供其它脚本注册。

然后根据请求检索到的 js 脚本，一股脑运行，让他们自己注册 hook。

接下来，在请求流程里，获取到上游优先级和排序之后，通过 js hook 重写排序。传给 js 包括请求的元数据（映射的 annotations 、渠道的 annotations、优先级、预排好的顺序、请求的 headers、完整的请求 body）这些信息。js 排好之后返回一个新数组，然后按这个新的顺序去请求。

然后在每次真正上游请求之前，再经过 js 的 hook ，决定要不要跳过这个请求，叫 before request hook 吧。这个 hook 接受和前面排序 hook 类似的输入，包括当前渠道的重试次数和总的尝试次数，历史尝试的错误信息，然后将返回 { next: false, delay: 3000 } 这样的结果。如果 next 是 true 则请求下一个 upstream ；如果是 false 则请求当前渠道。如果当前请求失败，则重试次数+1，并继续运行 hook ，这样可以让 js 自己根据错误决定是否重试。

接下来是重写请求的 hook，在上述 hook 结束后，对每个上游请求，调用重写 Hook，传入 headers, body, 和各类 annotations 元数据，重试次数啥的，然后再用重写后的请求发起真正请求。
